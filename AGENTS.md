# AGENTS.md — 本项目的 AI 协作规矩

## 三条开发规矩(用户明确指定,严格遵守)

### 1. 不自动同步 GitHub
- 修改代码后**不**主动执行 `git add` / `git commit` / `git push`
- 只有当用户明确说出"同步"、"提交"、"推送"、"上传"等指令时,才执行 Git 同步操作
- 同步完成后**不**自动验证 GitHub Pages 部署状态(除非用户要求)

### 2. 不每次做全面检查
- 修改完成后**只**对改动的文件做基本的 `node --check` 语法验证
- **不**跑整套端到端测试
- **不**读取与本次改动无关的文件来做交叉验证
- **不**主动检查其他模块的潜在 bug

### 3. 小范围改动
- 只动用户直接要求的内容
- **不**顺手"优化"无关模块
- **不**改 README、注释、格式、命名等用户未提及的内容
- **不**重构、不重写已有可工作的代码
- 改动跨多个文件时,先列出涉及文件,等用户确认后再动手

## 例外情况(必须先告诉用户,不自动做)
- 发现与本次需求直接相关的明显 bug → 提示但不修
- 改动必须跨多个文件才能完成 → 列出清单,等用户确认
- 用户明确要求"全面检查"、"跑测试"、"重构" → 才执行

## 默认工作流
1. 用户提出需求
2. 只改直接相关的代码,只 `node --check` 改动的文件
3. 告诉用户:改了什么、影响范围、不主动 git 同步
4. 等待用户指令再同步
